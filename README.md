# Campus Facility Booking System

Final-year DBMS project — **PostgreSQL 16 + FastAPI + psycopg2 (raw SQL, no ORM)**.
Students, faculty, clubs, and admins log in, browse available time slots on campus facilities, pay from a wallet, and complete bookings without colliding on the same slot. Correctness is pushed into the database via ACID transactions, CHECK constraints, triggers, and partial UNIQUE indexes.

## What's in the repo

| File / dir | Purpose |
|---|---|
| `schema.sql`           | 9 tables + 2 triggers + 5 indexes (authoritative schema) |
| `seed.py`              | Populates a fresh DB with 50 users, 15 facilities, 120 slots, sample bookings, and writes `credentials.txt` |
| `main.py`              | FastAPI backend (email login, bookings, wallet, admin endpoints) |
| `static/index.html`    | Single-page app (vanilla JS — Profile / Booking / Transactions / Admin) |

| `tests/`               | Pytest suite — 20+ bots, concurrency, rollback, direct-DB constraint checks |
| `TESTING.md`           | Step-by-step manual testing plan |
| `credentials.txt`      | Generated by `seed.py` — email + password for every seeded user (gitignored) |

## Quickstart (collaborator setup)

### 0. One-time prerequisites
- **Docker Desktop** (or Docker Engine + Compose plugin) — for Postgres/pgAdmin
- **Python 3.12+**
- **Git**

### 1. Clone
```bash
git clone <repo-url>
cd dbms_project
```

### 2. Start Postgres + pgAdmin
```bash
docker compose up -d
```
This applies `schema.sql` automatically the first time the `pgdata` volume is empty. Postgres runs on `:5432`, pgAdmin on `:5050` (login `admin@local.com` / `admin`).

### 3. Install Python deps
```bash
pip install -r requirements.txt
```
(Use a venv if you prefer: `python -m venv .venv && source .venv/bin/activate` first.)

### 4. Seed the database
```bash
python seed.py
```
This writes `credentials.txt` — the login list for the 50 seeded users.

### 5. Run the backend
```bash
python -m uvicorn main:app --port 8000 --reload
```
Open **http://localhost:8000** for the UI, or **http://localhost:8000/docs** for interactive API docs.

### 6. Run the automated tests
```bash
pytest -v tests/
```
See [tests/README.md](tests/README.md) for what each suite proves.

## Dev workflow

- **Watch every SQL statement the backend runs**
  ```bash
  docker compose logs -f postgres
  ```
- **Watch every HTTP request**: start uvicorn with `--log-level debug`.
- **Browse data visually**: pgAdmin at http://localhost:5050. Register a server with host `postgres`, port `5432`, db `campus_booking`, user `postgres`, password `postgres`.
- **Interactive API playground**: http://localhost:8000/docs (Swagger UI).
- **psql shell**:
  ```bash
  docker exec -it campus_pg psql -U postgres -d campus_booking
  ```
- **Reset DB state**: `python seed.py` truncates and re-seeds. Reproducible — passwords are deterministic per user_id.

## Architecture

- **Layer 1 — Identity & finance**: `Users`, `Students`, `Organizations`, `Wallet_Transactions` (immutable ledger).
- **Layer 2 — Inventory**: `Facilities`, `Facility_Rooms`, `Facility_Slots` (recurring timetable).
- **Layer 3 — Booking**: `Bookings` (cart), `Booking_Slots` (line items). Partial UNIQUE indexes enforce per-room and per-slot uniqueness even with NULLs.
- **Layer 4 — DB-enforced rules**:
  - `validate_ground_booking` trigger — only Organizations can book OAT/Pronite.
  - `prevent_double_booking` trigger — same slot/date/room can't collide, Cancelled carts ignored.
  - `CHECK` constraints — wallet never negative, password matches `^[A-Za-z]{4}[0-9]{4}$`, amounts > 0, etc.
  - `ON DELETE RESTRICT` on `Bookings.user_id` — history is immutable.
- **Layer 5 — Performance**: indexes on slot-by-facility, booking-date/status, ledger-by-user-time, etc.

Two kinds:

## Stopping / cleaning up

```bash
docker compose down            # stop containers, keep data
docker compose down -v         # stop AND wipe the pgdata volume (schema re-applies next time)
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| `connection refused on :5432` | `docker compose up -d`, wait 5s, retry |
| `credentials.txt not found` when running tests | `python seed.py` |
| API `500` errors right after seed | Schema may be stale — `docker compose down -v && docker compose up -d && python seed.py` |
| Tests fail with `API not reachable` | Start the backend: `python -m uvicorn main:app --port 8000` |
